# 📋 LINE 封鎖用戶追蹤系統 — 部署教學

## 📁 檔案說明
| 檔案 | 用途 |
|------|------|
| `app.py` | 主程式（Flask 伺服器） |
| `requirements.txt` | Python 套件清單 |
| `Procfile` | Railway 啟動設定 |

---

## 🚀 步驟一：上傳到 GitHub

1. 前往 https://github.com 並登入（沒帳號請先註冊）
2. 點右上角「+」→「New repository」
3. 取名為 `line-unfollow-tracker`，選 **Private（私有）**
4. 點「Create repository」
5. 將以上三個檔案上傳進去（點「uploading an existing file」）

---

## ☁️ 步驟二：部署到 Railway

1. 前往 https://railway.app 並用 GitHub 帳號登入
2. 點「New Project」→「」
3. 選擇你的 `line-unfollow-tracker` 儲存庫
4. Railway 會自動偵測並開始部署

### 設定環境變數
在 Railway 專案頁面：
1. 點「Variables」分頁
2. 新增以下變數：

| 變數名稱 | 值 |
|---------|-----|
| `LINE_CHANNEL_SECRET` | 你的 LINE Channel Secret |

### 取得部署網址
部署完成後，點「Settings」→「Domains」→「Generate Domain」
你會得到一個網址，例如：
```
https://line-unfollow-tracker-xxxx.railway.app
```

---

## 🔧 步驟三：設定 LINE Webhook

1. 前往 LINE Developers Console：https://developers.line.biz
2. 選擇你的 Messaging API Channel
3. 點「Messaging API」分頁
4. 找到「Webhook URL」欄位，填入：
```
https://你的Railway網址/webhook
```
5. 點「Verify」驗證（應該顯示 Success）
6. 開啟「Use webhook」開關

---

## 🖥️ 步驟四：查看後台

系統啟動後，你可以使用以下網址：

| 功能 | 網址 |
|------|------|
| 管理後台（看黑名單） | `https://你的網址/admin` |
| 匯出黑名單 JSON | `https://你的網址/export/blocked` |
| 系統狀態 | `https://你的網址/` |

## ❓ 常見問題

**Q: 之前被封鎖的人查得到嗎？**
A: 不行。LINE 不提供歷史封鎖資料，只能從部署後開始記錄。

**Q: Railway 免費嗎？**
A: Railway 有免費額度（每月 $5 美元的免費使用量），對這個小系統來說已經足夠。

**Q: 我要怎麼在發訊息前過濾黑名單？**
A: 呼叫 `/export/blocked` 取得封鎖名單，發訊息前先排除這些 user_id。
   如果你使用第三方群發工具，可以將 user_id 匯出給對方設定排除。
